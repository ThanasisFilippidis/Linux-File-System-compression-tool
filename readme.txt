An application which successfully accepts as input a path of a folder (even if that is the root folder) flattens its hierarchy,
compresses its subfolders and files while in the same time keeps intact the details of each file and has as an output a .di file
containing all that information. Moreover it can accept as an input a file of .di type, decompress it and place it wherever pointed 
successfully representing the initial hierarchy of all the folders and all the types of files.
